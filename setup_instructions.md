# Project Setup Guide

This is a local multiplayer game server built with **Node.js**, **TypeScript**, **Express**, and **PostgreSQL**. This guide walks you through getting the project running from scratch.

---

## Prerequisites

Make sure you have the following installed before continuing:

- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- [npm](https://www.npmjs.com/)
- [PostgreSQL](https://www.postgresql.org/) (running locally or accessible on your network)
- [ts-node](https://typestrong.org/ts-node/) (installed automatically via `npm install`)

---

## Project Structure

```
project-root/
├── src/
│   ├── server.ts        # App entry point
│   ├── db.ts            # PostgreSQL connection pool
│   └── routes/
│       ├── home.ts      # Home/lobby route
│       └── game.ts      # Game routes
├── dist/                # Compiled JS output (generated by build)
├── .env                 # Environment variables (you create this)
├── package.json
└── tsconfig.json
```

---

## Installation

Clone or download the project, then install dependencies:

```bash
npm install
```

---

## Environment Variables

Create a `.env` file in the **project root** with the following contents:

```env
# Server
PORT=3000

# PostgreSQL
DB_HOST=localhost
DB_PORT=5432
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_NAME=your_db_name
```

> Replace the `DB_*` values with your actual PostgreSQL credentials.

---

## Database Setup

Make sure your PostgreSQL server is running and that the database specified in `DB_NAME` exists. You can create it with:

```sql
CREATE DATABASE your_db_name;
```

The app will verify the connection on startup and exit with an error if it cannot reach the database.

---

## Running the Project

### Development (with hot reload)

```bash
npm run dev
```

This uses `nodemon` and `ts-node` to watch for file changes and restart automatically.

### Production

First compile TypeScript to JavaScript:

```bash
npm run build
```

Then start the compiled server:

```bash
npm start
```

## API Endpoints

| Method | Route          | Description                              |
|--------|----------------|------------------------------------------|
| GET    | `/ping`        | Health check — confirms the server is up |
| GET    | `/home/home`   | Returns lobby welcome message            |
| GET    | `/game/status` | Checks DB connection and game readiness  |
| POST   | `/game/join`   | Placeholder for a second player joining  |

### Example: Health Check

```bash
curl http://localhost:3000/ping
```

```json
{ "message": "Server is alive!" }
```

### Example: Join Game

```bash
curl -X POST http://localhost:3000/game/join \
  -H "Content-Type: application/json" \
  -d '{"playerName": "Alice"}'
```

```json
{ "message": "Player Alice is attempting to join the host." }
```

## LAN Multiplayer

The server binds to `0.0.0.0`, which means other machines on the same local network can connect to it. A second player can reach the server at:

```
http://<host-machine-ip>:<PORT>
```

To find your machine's local IP:

- **Windows:** `ipconfig` in Command Prompt
- **macOS/Linux:** `ifconfig` or `ip a` in Terminal
